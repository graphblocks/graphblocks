# Roadmap

Near-term priorities are:

1. Reconcile monorepo package versions, dependency ranges, metadata, and
   publishable artifacts.
2. Close Python/Rust parity gaps for orchestration limits, governed workspace
   commits, deployment evidence, telemetry outbox behavior, webhook delivery,
   and provider-authoritative voice.
3. Add restart-durable accepted-run checkpoint recovery and remote-worker claim
   semantics to a durable runtime implementation.
4. Stabilize diagnostics, versioning, compatibility policy, and release
   evidence for a first public alpha release.
5. Expand adapter integration tests without adding heavy dependencies to the
   default authoring package.

Roadmap items are non-normative. A feature becomes supported only when its
specification, implementation, fixtures, and required acceptance evidence agree.
