# Example Applications

각 YAML은 parser가 읽을 수 있는 의사 코드이며, 현재 공식 implementation이 존재한다는 뜻은 아니다.

1. `01-enterprise-federated-rag.yaml`: 복합 RAG와 named resource binding
2. `02-document-ingestion.yaml`: per-item checkpoint와 staged ingestion
3. `03-policy-governed-chat.yaml`: graceful turn completion과 hard stop
4. `04-tui-workspace-assistant.yaml`: TUI를 graph node가 아닌 Application Protocol client로 모델링
5. `05-authority-backed-advisory.yaml`: 한국 법률 API를 fixture로 사용한 source/evidence/review 일반화
6. `06-bounded-research-orchestrator.yaml`: bounded TaskPlan 기반 research orchestration
7. `07-verified-rtl-workspace-trial.yaml`: Verilog 수정/검증을 fixture로 사용한 ChangeSet/Check/Gate/Lease 일반화
8. `08-kubernetes-production-deployment.yaml`: immutable release, target, placement, canary, drain
9. `09-observability-profile.yaml`: telemetry와 durable records 분리
10. `10-realtime-voice-extension.yaml`: 선택적 duplex voice extension
11. `11-coding-agent-background-callbacks.yaml`: background coding agent run, cursor replay,
    callback subscription, and authenticated external CI callback flow
